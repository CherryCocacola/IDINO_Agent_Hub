## H10 핫픽스 효과 상세

### 문제 (이전)
`POST /reports/generate` 에 `template_id` = `tb_document_templates`의 UUID를 전달하면
ORM이 이를 `tb_generated_reports.template_id` (FK → `tb_report_templates`)에 직접 저장 시도.
실제로는 `tb_report_templates`에 없는 UUID이므로 PostgreSQL FK 위반 → HTTP 500

### 수정 (H10)
`reports/service.py`의 `generate_report()` 메서드에서:
1. `data.template_id`를 먼저 `tb_report_templates`에서 조회 (NOT FOUND → `tb_document_templates` 조회)
2. `tb_document_templates`에 존재하면:
   - `generation_params["document_template_id"] = str(data.template_id)` 로 이전
   - `orm_template_id = None` (FK 컬럼은 NULL로)
3. FK 위반 없이 INSERT 성공

### 검증 결과 (API 응답으로 확인)
```json
{
  "template_id": null,
  "generation_params": {
    "document_template_id": "dcfbd218-9cf1-405e-b703-f652c202aaca",
    "prompt": "...",
    "_session_user": {...}
  },
  "rendering_mode": "jinja2",
  "status": "completed"
}
```
