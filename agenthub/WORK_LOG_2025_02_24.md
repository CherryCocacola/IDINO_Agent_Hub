## 1. 할당량 관리 및 모니터링 조회 수정 ✅

**수정 내용**:

**파일**: `ClientApp/src/views/Quota.vue`

## 2. 감사 로그 동작 수정 ✅

**문제**: `loadLogs()`가 API를 호출하지 않고 빈 배열만 반환

**수정 내용**:
- **ActivityLogController.cs** 신규 생성: `/api/activitylog` GET 엔드포인트
- **ActivityLogService.cs** 신규 생성: DB `ActivityLogs` 테이블 조회, 필터(날짜, 사용자, 활동 유형, 검색) 지원
- **ActivityLogDto.cs** 신규 생성
- **Program.cs**: `IActivityLogService`, `ActivityLogService` DI 등록
- **AuditLog.vue**: `loadLogs()`에서 `/api/activitylog` 호출, 서버 사이드 페이지네이션 적용, `totalCount` 기반 페이지 수 계산
- 검색/필터 적용 시 `loadLogs()` 호출, 페이지 이동 시 `loadLogs()` 호출

**파일**: 
- `Controllers/ActivityLogController.cs` (신규)
- `Services/ActivityLogService.cs` (신규)
- `Services/IActivityLogService.cs` (신규)
- `DTOs/ActivityLogDto.cs` (신규)
- `Program.cs`
- `ClientApp/src/views/AuditLog.vue`

## 3. 사용 기록 동작 수정 ✅

**문제**: 페이지네이션 및 검색이 제대로 동작하지 않음

**수정 내용**:
- **AnalyticsService**: `GetUsageHistoryAsync` 반환 타입을 `(List<ApiUsageDto>, int TotalCount)`로 변경
- **AnalyticsController**: `usage-history` 응답을 `{ items, totalCount }` 형태로 변경
- **search** 파라미터 추가: 사용자명, 서비스명, 모델명으로 서버 사이드 검색
- **UsageHistory.vue**: 
  - `totalUsageCount` ref 추가
  - API 응답에서 `items`, `totalCount` 파싱
  - `totalPages`를 `totalUsageCount` 기반으로 계산
  - 검색 시 `params.search` 전달
  - `loadStats`에서 `take: 10000`으로 통계용 데이터 로드

**파일**: 
- `Services/AnalyticsService.cs`
- `Services/IAnalyticsService.cs`
- `Controllers/AnalyticsController.cs`
- `ClientApp/src/views/UsageHistory.vue`

## 4. 프레젠테이션 PDF 정상화 ✅

**문제**: LibreOffice 미설치 시 PDF 내보내기 실패

**수정 내용**:
- **PresentationController**: PDF 변환 실패 시 PPTX로 자동 폴백

  - `X-Pdf-Fallback-Pptx: true` 헤더 추가
- **PresentationBuilder.vue**: 해당 헤더 확인 시 `.pptx` 확장자로 다운로드, 콘솔 안내 메시지 출력

**파일**: 
- `Controllers/PresentationController.cs`
- `ClientApp/src/views/PresentationBuilder.vue`

## 5. 프레젠테이션 저장 오류 수정 (문자열 잘림) ✅

**문제**: "Failed to save presentation: 문자열이나 이진 데이터는 잘립니다" 오류

**수정 내용**:
- **PresentationService**:
  - 첫 저장 시 `NormalizeSlidesForSerialization`에 `maxContentPerSlide: 20000` 적용
  - `DefaultMaxContent` 50000 → 20000으로 축소
  - JSON 크기 500,000자 초과 시 `TruncateSlidesToFitMaxSize`로 400,000자 이하로 사전 절단
  - 재시도 시 `maxContentPerSlide: 5000`, `SafeMaxChars: 8000` 사용

**파일**: `Services/PresentationService.cs`

## 6. 슬라이드 본문 Markdown 편집기 및 그래프/표 미리보기 ✅

**수정 내용**:
- **marked**, **DOMPurify** 사용해 본문 Markdown 렌더링
- `renderMarkdown()` 함수 추가 (XSS 방지)
- **편집기**: 본문 라벨에 "(Markdown 지원)" 표시, 사용 안내 문구 추가
- **미리보기**: 
  - 본문/2단 콘텐츠에 `v-html="renderMarkdown(...)"` 적용
  - `tables` 배열이 있으면 HTML `<table>`로 렌더링
  - `charts` 배열이 있으면 제목 + 데이터 테이블로 렌더링
- **슬라이드쇼**에도 동일한 Markdown/표/차트 렌더링 적용
- **CSS**: `.pbr-prev-markdown`, `.pbr-prev-tables`, `.pbr-prev-chart` 스타일 추가

**파일**: 
- `ClientApp/src/views/PresentationBuilder.vue`
- `ClientApp/src/views/PresentationBuilder.css`
- `ClientApp/src/types/index.ts` (TableDto 추가)

- **감사 로그**: Admin 역할 필요. `ActivityLogs` 테이블에 데이터가 있어야 목록이 표시됩니다.
