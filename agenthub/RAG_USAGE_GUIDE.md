# RAG (Retrieval-Augmented Generation) 사용 가이드

## 개요
RAG 기능은 Knowledge Base에 저장된 문서를 검색하여 AI 답변의 정확성과 신뢰성을 향상시키는 기능입니다.

## 사용 방법

### 1단계: Knowledge Base에 문서 추가

#### 방법 1: 새 문서 작성
1. 좌측 메뉴에서 **지식베이스** 클릭
2. 우측 상단의 **"새 문서 작성"** 버튼 클릭
3. 문서 제목과 내용 입력
4. **"저장"** 버튼 클릭
   - 저장 시 자동으로 인덱싱됨 (`indexImmediately: true`)

#### 방법 2: 파일 업로드
1. 우측 상단의 **"파일 업로드"** 버튼 클릭
2. 파일 선택 (PDF, TXT, CSV, XLSX, DOCX 지원)
3. 제목 입력 (선택사항, 기본값: 파일명)
4. **"업로드 후 즉시 인덱싱"** 체크박스 선택 (권장)
5. **"업로드"** 버튼 클릭
   - 파일이 파싱되어 Knowledge Base에 저장됨
   - 선택한 경우 자동으로 인덱싱됨

### 2단계: 문서 인덱싱 확인/수동 인덱싱

#### 인덱싱 상태 확인
- 문서 목록에서 각 문서의 인덱싱 상태 확인:
  - 🟢 **인덱싱됨**: RAG 검색에 사용 가능
  - 🟡 **미인덱싱**: RAG 검색에 사용 불가 (인덱싱 필요)

#### 수동 인덱싱
1. 문서 목록에서 해당 문서 클릭
2. 문서가 선택되면 상단 툴바에 **"인덱싱"** 또는 **"재인덱싱"** 버튼 표시
3. 버튼 클릭하여 인덱싱 수행
   - **인덱싱**: 처음 인덱싱하는 경우
   - **재인덱싱**: 문서 내용을 수정한 후 다시 인덱싱하는 경우

> **참고**: 인덱싱은 문서를 작은 청크(Chunk)로 분할하고, 각 청크에 대한 임베딩(벡터)을 생성하여 저장합니다. 이 과정은 시간이 소요될 수 있습니다.

### 3단계: 채팅에서 RAG 활성화

#### AgentChat (단일 채팅)
1. 좌측 사이드바의 **"모델 설정"** 섹션에서
2. **"RAG 활성화 (Knowledge Base)"** 체크박스 활성화
3. 또는 **"고급 설정"** 모달에서 활성화

#### AgentMultiChat (멀티 채팅)
1. 채팅 헤더의 **설정 버튼** (⚙️) 클릭
2. 설정 모달에서 **"RAG 활성화 (Knowledge Base)"** 체크박스 활성화
3. **"저장"** 버튼 클릭

### 4단계: 질문하기

1. RAG가 활성화된 상태에서 질문 입력
2. AI는 다음 과정을 거쳐 답변합니다:
   - 사용자 질문을 임베딩 벡터로 변환
   - Knowledge Base에서 유사한 문서 청크 검색 (코사인 유사도 기반)
   - 검색된 관련 컨텍스트를 AI 시스템 메시지에 추가
   - 컨텍스트를 기반으로 더 정확한 답변 생성

## 주요 기능 설명

### 인덱싱 필터
- 문서 목록 상단의 필터 버튼:
  - **전체**: 모든 문서 표시
  - **인덱싱됨** (🟢): 인덱싱된 문서만 표시
  - **미인덱싱** (🟡): 인덱싱되지 않은 문서만 표시

### 검색 기능
- 우측 상단의 검색 창에서 문서 제목/내용으로 검색
- 검색 결과에서 원하는 문서 선택

### 문서 정보
- 우측 패널에서 선택한 문서의 정보 확인:
  - 생성일, 수정일
  - 인덱싱일
  - 청크 수 (인덱싱된 청크 개수)
  - 소스 타입

## 설정 옵션

### RAG Top-K
- RAG 검색 시 가져올 최대 문서 청크 개수
- 기본값: 5개
- 값이 클수록 더 많은 컨텍스트를 제공하지만, 토큰 사용량이 증가

### RAG와 웹 검색 동시 사용
- RAG 활성화와 웹 검색(Tavily)을 동시에 활성화할 수 있음
- RAG: Knowledge Base 문서 검색
- 웹 검색: 최신 웹 정보 검색
- 두 기능을 함께 사용하면 더 풍부한 컨텍스트 제공

## 주의사항

1. **인덱싱 필요**: RAG를 사용하려면 문서가 인덱싱되어 있어야 합니다.
2. **사용자별 문서**: 각 사용자는 자신이 업로드/생성한 문서만 검색할 수 있습니다.
3. **인덱싱 시간**: 대용량 문서의 경우 인덱싱에 시간이 소요될 수 있습니다.
4. **토큰 사용량**: RAG를 사용하면 검색된 컨텍스트가 시스템 메시지에 추가되므로 토큰 사용량이 증가할 수 있습니다.

## 문제 해결

### RAG가 작동하지 않는 경우
1. Knowledge Base에 문서가 있는지 확인
2. 문서가 인덱싱되어 있는지 확인 (🟢 인덱싱됨 표시)
3. RAG 활성화 체크박스가 켜져 있는지 확인
4. 문서 내용과 질문이 관련이 있는지 확인

### 인덱싱이 실패하는 경우
1. 문서 내용이 비어있지 않은지 확인
2. OpenAI API 키가 올바르게 설정되어 있는지 확인
3. 네트워크 연결 상태 확인
4. 서버 로그에서 에러 메시지 확인

## 예시 시나리오

### 시나리오 1: 회사 정책 문서 기반 질문
1. 회사 정책 PDF 파일을 Knowledge Base에 업로드
2. 자동 인덱싱 완료 대기
3. 채팅에서 RAG 활성화
4. "연차 사용 정책은 어떻게 되나요?" 질문
5. AI가 업로드된 정책 문서를 참조하여 정확한 답변 제공

### 시나리오 2: 기술 문서 기반 질문
1. 기술 문서를 Knowledge Base에 작성
2. 수동으로 인덱싱 수행
3. 채팅에서 RAG 활성화
4. 기술 관련 질문
5. AI가 기술 문서의 내용을 참조하여 답변

## API 엔드포인트 (참고)

- `GET /api/knowledgebase` - 문서 목록 조회
- `GET /api/knowledgebase/{id}` - 문서 상세 조회
- `POST /api/knowledgebase` - 문서 생성
- `PUT /api/knowledgebase/{id}` - 문서 수정
- `DELETE /api/knowledgebase/{id}` - 문서 삭제
- `POST /api/knowledgebase/{id}/index` - 문서 인덱싱
- `POST /api/knowledgebase/{id}/reindex` - 문서 재인덱싱
- `POST /api/files/upload/knowledgebase` - 파일 업로드 및 자동 인덱싱
