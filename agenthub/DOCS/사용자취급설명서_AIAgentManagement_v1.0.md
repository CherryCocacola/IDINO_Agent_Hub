## 1. 제품 개요

### 1.1 개요

AI Agent 통합관리 시스템(AIAgentManagement)은 기관·기업 내 다양한 인공지능(AI) 서비스를 단일 플랫폼에서 통합 운영·관리할 수 있도록 설계된 웹 기반 SaaS형 소프트웨어입니다.

본 시스템은 OpenAI(GPT), Anthropic(Claude), Google(Gemini), Perplexity, Mistral 등 주요 생성형 AI API를 하나의 인터페이스로 연결하고, 사용 목적에 맞는 AI Agent를 생성·배포·관리하는 기능을 제공합니다.

별도의 소프트웨어 설치 없이 웹 브라우저만으로 모든 기능을 이용할 수 있으며, 역할 기반 접근 제어(RBAC)를 통해 관리자·일반 사용자 간 권한을 명확히 분리합니다.

### 1.2 용도 및 목적

| 구분 | 내용 |
|---|---|
| **주요 용도** | 기관·기업 내 AI 서비스 통합 운영 플랫폼 |
| **목적** | 다양한 AI API를 단일 인터페이스로 관리하여 업무 효율화 및 AI 활용 비용 최적화 |
| **주요 사용자** | AI 서비스 운영 담당자, 일반 사용자, 시스템 관리자 |
| **도입 효과** | AI 서비스 도입 비용 절감, 조직 내 AI 활용 표준화, 사용량 모니터링 및 통제 |

**주요 활용 시나리오:**
- 부서별 전용 AI Agent 생성 및 배포 (인사, 법무, 고객응대 등)
- 사내 문서를 기반으로 한 AI 질의응답 서비스 구축 (RAG)
- 반복 업무 자동화를 위한 워크플로우 설계
- AI 이미지 생성, 프레젠테이션 자동 생성 등 창작 업무 지원
- 외부 서비스에 AI Agent 임베드(Embed) 제공

#### 2.1.2 데이터베이스 생성

```sql

CREATE DATABASE AIAgentManagement;

-- 2. 전용 계정 생성 및 권한 부여
CREATE LOGIN aiuser WITH PASSWORD = '강력한비밀번호';
USE AIAgentManagement;
CREATE USER aiuser FOR LOGIN aiuser;
ALTER ROLE db_owner ADD MEMBER aiuser;
```

#### 2.1.3 DB 스키마 생성 (마이그레이션)

배포 파일이 있는 폴더에서 다음 명령을 실행합니다.

```powershell
# Entity Framework Core 마이그레이션 적용
dotnet AIAgentManagement.dll -- --migrate

# 또는 SQL 스크립트 방식 (제공된 경우)
sqlcmd -S 서버명 -d AIAgentManagement -i migration.sql
```

#### 2.1.4 환경변수 설정

관리자 권한 PowerShell에서 제공된 설정 스크립트를 실행합니다.

```powershell
# 관리자 권한으로 실행
Set-ExecutionPolicy RemoteSigned -Scope Process
.\iis-setting.ps1
```

스크립트가 설정하는 환경변수 목록:

| 환경변수명 | 설명 | 예시 |
|---|---|---|
| `ConnectionStrings__DefaultConnection` | SQL Server 연결 문자열 | `Server=...;Database=AIAgentManagement;...` |
| `JwtSettings__SecretKey` | JWT 서명 키 (32자 이상) | *(비밀값)* |
| `AiApiSettings__OpenAI__ApiKey` | OpenAI API 키 | `sk-...` |
| `AiApiSettings__Claude__ApiKey` | Anthropic API 키 | `sk-ant-...` |
| `AiApiSettings__Gemini__ApiKey` | Google Gemini API 키 | *(비밀값)* |
| `EmailSettings__SmtpPassword` | 이메일 발송 비밀번호 | *(비밀값)* |

> **보안 주의**: API 키 등 민감한 값은 appsettings.json에 직접 기재하지 않고 반드시 환경변수(Machine 범위)로 설정합니다.

### 2.3 제품의 사용방법

#### 2.3.1 회원가입 및 로그인

**회원가입 절차**

1. 서비스 URL 접속 → **[시작하기]** 또는 **[회원가입]** 버튼 클릭
2. 이메일 주소 입력 (중복 불가)
3. 비밀번호 입력 (최소 8자 이상)
4. 이름, 부서(선택) 입력
5. 이용약관 및 개인정보처리방침 동의 체크
6. **[회원가입]** 버튼 클릭
7. 관리자 승인 후 계정 활성화 (Status: Pending → Active)

> **참고**: 회원가입 후 관리자가 계정을 활성화해야 로그인이 가능합니다.

**로그인 절차**

1. 서비스 URL 접속 → **[로그인]** 버튼 클릭
2. 이메일 주소 및 비밀번호 입력
3. **[로그인]** 버튼 클릭
4. 로그인 성공 시 대시보드 화면으로 이동

**비밀번호 재설정**

1. 로그인 화면에서 **[비밀번호를 잊으셨나요?]** 클릭
2. 가입 이메일 주소 입력
3. **[재설정 링크 발송]** 클릭
4. 수신된 이메일의 링크 클릭 (유효기간 2시간)
5. 새 비밀번호 입력 및 확인 후 저장

#### 2.3.3 AI Agent 생성 및 관리

**Agent 생성 절차**

1. 좌측 메뉴 → **[Agent 관리]** → **[새 Agent 만들기]** 클릭
2. 기본 정보 입력:
   - **Agent 이름** (필수): 식별 가능한 명칭 입력
   - **설명** (선택): Agent 역할 및 용도 설명
   - **아이콘/색상** (선택): 구분을 위한 시각적 표시
3. AI 서비스 설정:
   - **AI 서비스** (필수): OpenAI, Claude, Gemini 등 선택
   - **모델** (필수): 선택한 서비스의 사용 모델 선택
   - **Temperature** (0.0~2.0): 응답 창의성 조절 (기본: 0.7)
   - **Max Tokens** (1~128,000): 최대 응답 길이 설정
4. 시스템 프롬프트 입력: Agent의 역할·페르소나·지시사항 작성
5. 공개 설정:
   - **비공개**: 본인만 사용
   - **공개**: 모든 로그인 사용자 사용 가능
   - **게스트 채팅 허용**: 비로그인 게스트 채팅 허용 여부
6. **[저장]** 클릭

**Agent 수정/삭제**

- Agent 목록에서 해당 Agent 선택 → **[수정]** 또는 **[삭제]**
- 삭제 시 확인 다이얼로그 표시

**Agent 임베드 코드 생성**

1. Agent 상세 화면 → **[임베드]** 탭
2. 허용 도메인 입력 (예: `https://mycompany.com`)
3. 생성된 `<script>` 코드를 외부 웹페이지에 삽입

**QR 코드 생성**

1. 공개 Agent 상세 화면 → **[QR 코드]** 탭
2. QR 코드 이미지 다운로드 또는 복사

#### 2.3.4 AI 채팅

**단일 Agent 채팅**

1. 좌측 메뉴 → **[채팅]**
2. **[새 대화]** 버튼 클릭
3. 사용할 Agent 선택
4. 메시지 입력 후 **[전송]** (또는 Enter)
5. AI 응답이 실시간 스트리밍으로 표시됨
6. 대화는 자동 저장됨

**파일 첨부 채팅**

1. 채팅 입력창 좌측 **[파일 첨부]** 아이콘 클릭

3. 파일과 함께 질문 메시지 입력
4. 전송 — AI가 파일 내용을 분석하여 응답

**멀티 에이전트 채팅**

1. 좌측 메뉴 → **[멀티 채팅]**
2. 비교할 Agent 선택 (최대 4개)
3. 메시지 입력 후 전송
4. 선택된 모든 Agent의 응답이 나란히 표시됨

**대화 관리**

| 기능 | 방법 |
|---|---|
| 대화 검색 | 채팅 목록 상단 검색창에 키워드 입력 |
| 대화 이름 변경 | 대화 목록에서 우클릭 → [이름 변경] |
| 대화 핀 고정 | 우클릭 → [핀 고정] |
| 대화 아카이브 | 우클릭 → [아카이브] |
| 대화 삭제 | 우클릭 → [삭제] → 확인 |

#### 2.3.9 API 키 관리

**API 키 발급**

1. 우측 상단 프로필 → **[API 키 관리]**
2. **[새 API 키 발급]** 클릭
3. 키 이름(식별용), 권한 범위(Scope) 선택:
   - `stream`: 채팅 스트리밍 API 접근
   - `info`: Agent 정보 조회 API 접근
   - `usage`: 사용량 조회 API 접근

5. **[발급]** 클릭
6. 발급된 API 키 복사 (이후 재확인 불가)

**API 키 사용 예시**

```http
POST /v1/chat/completions
Authorization: Bearer {발급된 API 키}
Content-Type: application/json

{
  "model": "gpt-4",
  "messages": [{"role": "user", "content": "안녕하세요"}]
}
```

#### 2.4.1 데이터베이스 백업

**방법 1 — 관리자 화면에서 수동 백업**

1. **[관리자]** → **[DB 백업]**
2. **[백업 실행]** 버튼 클릭
3. 백업 완료 후 파일 경로 확인

**방법 2 — SQL Server Agent 자동 백업 (권장)**

SQL Server Management Studio에서 유지 관리 계획(Maintenance Plan)을 생성하여 매일 자정 자동 백업을 설정합니다.

```sql
-- 수동 백업 SQL 명령 예시
BACKUP DATABASE AIAgentManagement
TO DISK = 'C:\Backup\AIAgentManagement_20260318.bak'
WITH FORMAT, MEDIANAME = 'AIAgentManagement', NAME = 'Full Backup';
```

**업로드 파일 백업**

```powershell
# wwwroot/uploads 폴더를 외부 스토리지에 동기화
robocopy C:\inetpub\wwwroot\aigenthub\uploads D:\Backup\uploads /MIR
```

#### 2.4.2 복구 절차

데이터 손실 또는 서버 장애 발생 시 다음 절차를 수행합니다.

1. **DB 복원**
```sql
RESTORE DATABASE AIAgentManagement
FROM DISK = 'C:\Backup\AIAgentManagement_20260318.bak'
WITH REPLACE, RECOVERY;
```

2. **업로드 파일 복원**
```powershell
robocopy D:\Backup\uploads C:\inetpub\wwwroot\aigenthub\uploads /MIR
```

3. **환경변수 확인** — `iis-setting.ps1` 재실행 또는 Machine 환경변수 수동 확인

4. **애플리케이션 재시작**
```powershell
iisreset
```

5. **정상 동작 확인** — 브라우저에서 서비스 접속 및 로그인 테스트

### 2.6 외부 시스템(SW)과의 연동

#### 2.6.1 AI API 연동

본 제품은 다음 외부 AI API와 연동됩니다.

| 서비스 | 공급사 | 용도 | 연동 방식 |
|---|---|---|---|
| OpenAI API | OpenAI | GPT 채팅, DALL-E 이미지 생성, 텍스트 임베딩 | REST API (HTTPS) |
| Anthropic Claude API | Anthropic | Claude 모델 채팅 | REST API (HTTPS) |
| Google Gemini API | Google | Gemini 모델 채팅, 멀티모달 | REST API (HTTPS) |
| Perplexity API | Perplexity AI | 웹 검색 기반 AI 채팅 | REST API (HTTPS) |
| Mistral API | Mistral AI | Mistral 모델 채팅 | REST API (HTTPS) |
| Tavily API | Tavily | AI 웹 검색 | REST API (HTTPS) |

**AI API 키 등록 방법**

1. 해당 AI 서비스 공식 사이트에서 API 키 발급
2. **[관리자]** → **[AI 서비스 설정]** → 해당 서비스 클릭
3. API 키 입력 → **[저장]**
4. 서비스 활성화 토글 ON

#### 2.6.2 이메일 서버(SMTP) 연동

비밀번호 재설정 이메일 발송을 위해 SMTP 서버 설정이 필요합니다.

**설정 방법 (환경변수)**

```powershell
[System.Environment]::SetEnvironmentVariable("EmailSettings__SmtpServer", "smtp.gmail.com", "Machine")
[System.Environment]::SetEnvironmentVariable("EmailSettings__SmtpPort", "587", "Machine")
[System.Environment]::SetEnvironmentVariable("EmailSettings__SmtpUser", "your@gmail.com", "Machine")
[System.Environment]::SetEnvironmentVariable("EmailSettings__SmtpPassword", "앱비밀번호", "Machine")
[System.Environment]::SetEnvironmentVariable("EmailSettings__FromEmail", "your@gmail.com", "Machine")
```

#### 2.6.3 Redis 연동 (선택)

Redis는 세션 캐싱 및 분산 캐시 용도로 사용됩니다.

```powershell
# Redis 연결 문자열 설정
[System.Environment]::SetEnvironmentVariable("ConnectionStrings__Redis", "localhost:6379", "Machine")
```

Redis 미설치 시 In-Memory 캐시로 자동 대체됩니다.

#### 2.6.4 LibreOffice 연동 (선택)

HWP 문서 텍스트 추출 및 PPTX→PDF 변환에 LibreOffice가 필요합니다.

```powershell
# LibreOffice 설치 경로 설정
[System.Environment]::SetEnvironmentVariable("LibreOfficePath", "C:\Program Files\LibreOffice\program\soffice.exe", "Machine")
```

#### 2.6.5 OpenAI 호환 API 연동

기존 OpenAI SDK를 사용하는 외부 서비스와의 호환을 위해 표준 엔드포인트를 제공합니다.

| 항목 | 내용 |
|---|---|
| 엔드포인트 | `POST /v1/chat/completions` |
| 인증 | `Authorization: Bearer {API 키}` |
| 요청 형식 | OpenAI Chat Completions API와 동일 |
| 스트리밍 | `"stream": true` 지원 |

**사용 예시 (Python)**

```python
from openai import OpenAI

client = OpenAI(
    api_key="발급받은_API_키",
    base_url="https://agenthub.idino.co.kr"
)

response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "안녕하세요"}]
)
```

#### 2.6.6 Swagger API 문서

서비스 내 REST API 전체 명세를 Swagger UI에서 확인할 수 있습니다.

- URL: `https://agenthub.idino.co.kr/swagger`
- 인증: JWT Bearer Token (로그인 후 토큰 입력)
